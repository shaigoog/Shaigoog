# سجل شيقوق — متطلبات النسخة M3

## 1) العملات
- الدرهم الإماراتي — AED
- الدولار الأمريكي — USD
- الجنيه السوداني — SDG
- الجنيه المصري — EGP
- الريال السعودي — SAR
- اختيار العملة داخل كل صفقة ودفعة ومصروف.
- عدم جمع العملات المختلفة في رقم واحد داخل الداشبورد.
- عرض إجمالي مستقل لكل عملة.
- حفظ المبالغ كأرقام صحيحة في أصغر وحدة نقدية، بدون Float أو Double.

## 2) الأوزان والوحدات
- طن.
- كيلوجرام.
- جوال/شوال.
- عند اختيار الجوال: إدخال عدد الجوالات ووزن الجوال بالكيلوجرام.
- حساب الوزن الإجمالي بالكيلوجرام والطن تلقائياً.
- عرض الكمية بالوحدة الأصلية وبالطن المكافئ.

## 3) الداشبورد والتصميم
- تطوير الداشبورد مع الحفاظ على الفكرة الأساسية الحالية.
- ألوان أساسية: كحلي داكن، ذهبي هادئ، أبيض وخلفيات رمادية فاتحة.
- بطاقات واضحة ومناسبة لشاشة الهاتف بدون سحب أفقي.
- مؤشرات منفصلة:
  - عدد الجهات.
  - عدد الأصناف.
  - عدد الصفقات.
  - إجمالي الوزن.
  - إجمالي الشراء والبيع لكل عملة.
  - الصفقات الأخيرة.
- أزرار سريعة: إضافة جهة، إضافة صنف، إضافة صفقة.

## 4) النسخ الاحتياطي اليدوي
- إنشاء نسخة احتياطية يدوياً فقط.
- تصدير ملف واحد بصيغة ZIP يحتوي على:
  - البيانات.
  - المرفقات.
  - معلومات الإصدار وتاريخ النسخة.
- اختيار مكان حفظ الملف من مدير ملفات الهاتف.
- استعادة النسخة من ملف يختاره المستخدم.
- فحص سلامة النسخة قبل الاستعادة.
- طلب البصمة قبل الاستعادة أو استبدال البيانات الحالية.

## 5) المستندات والمرفقات
- إرفاق مستند بالصفقة أو الجهة.
- دعم PDF وJPG وJPEG وPNG.
- اختيار المستند من الهاتف.
- تصوير مستند بالكاميرا.
- نسخ المرفق إلى مساحة التطبيق الداخلية حتى لا يضيع عند نقل الملف الأصلي.
- عرض اسم الملف ونوعه وتاريخ الإضافة.
- فتح أو حذف المرفق.
- طلب البصمة قبل حذف المرفق.

## 6) رقم الهاتف وجهات الاتصال
- زر لاختيار الرقم من جهات اتصال الهاتف.
- تعبئة اسم الجهة ورقم الهاتف تلقائياً قدر الإمكان.
- السماح بتعديل الرقم بعد اختياره.
- زر اتصال مباشر.
- زر فتح محادثة واتساب.
- عدم إرسال أي رسالة تلقائياً.

## 7) البصمة والأمان
- طلب البصمة عند فتح التطبيق.
- طلب البصمة قبل:
  - تعديل سجل موجود.
  - حذف سجل أو مرفق.
  - استعادة نسخة احتياطية.
  - مسح جميع البيانات.
- السماح باستخدام رمز قفل الهاتف كبديل عندما يدعمه الجهاز.
- قفل التطبيق تلقائياً بعد بقائه في الخلفية فترة محددة.

## 8) معايير القبول قبل تسليم APK
- تثبيت التطبيق وفتحه بنجاح.
- إضافة جهة من جهات الاتصال.
- إضافة صنف.
- تسجيل صفقة بعملة ووحدة وزن مختلفتين.
- إغلاق التطبيق وفتحه والتأكد من بقاء البيانات.
- إنشاء نسخة احتياطية واستعادتها بنجاح.
- إرفاق وفتح مستند.
- نجاح البصمة عند الفتح والحذف والاستعادة.
- نجاح GitHub Actions وإنتاج APK تشغيلي.

> ملاحظة: تم اعتبار الكلمة الأخيرة «سوداني» في طلب العملات مقصوداً بها «الريال السعودي». يتم تعديلها إذا كان المقصود مختلفاً.
